# Nythos Cyber Aquarium — Windows Screensaver
"Every security journey begins with a single ripple." 🐬
A native **C++ / DirectX 11** Windows screensaver: a living reef aquarium that
doubles as a calm cybersecurity dashboard. Autonomous fish, god rays, caustics,
rising bubbles, HDR bloom, a semi-transparent cyber HUD fed by **real** system
telemetry, a bottom-right clock, and aquatic-themed security tips generated by a
**local Ollama** model (with a full offline fallback).

Builds to a true `NythosCyberAquarium.scr` supporting `/s`, `/p`, `/c`,
multi-monitor, and High-DPI.

## Build

Requirements: Visual Studio 2022/2026 with the **Desktop C++** workload
(MSVC + Windows 10/11 SDK). No other dependencies — shaders compile at runtime,
the CRT is linked statically, and the `.scr` is self-contained.

```powershell
pwsh -ExecutionPolicy Bypass -File build.ps1
# -> build\NythosCyberAquarium.scr
```

`build.ps1` embeds `shaders/*.hlsl` into `src/Shaders.gen.h`, compiles the
resource script, then compiles + links all of `src/*.cpp`.

## Install / Run

```powershell
.\install.ps1            # copy to System32 + open Screen Saver settings (needs admin)
.\install.ps1 -Preview   # run full-screen right now, no install
.\install.ps1 -Config    # open the settings dialog
```

Or right-click `build\NythosCyberAquarium.scr` in Explorer → **Install**.

### Command-line switches (the standard screensaver contract)
| Switch | Meaning |
|--------|---------|
| `/s` | Run full-screen on every monitor; exits on any mouse/keyboard input. |
| `/p <hwnd>` | Render the small preview inside the Settings window `<hwnd>`. |
| `/c[:<hwnd>]` | Show the configuration dialog. |
| *(none)* | Treated as `/c`. |

## Architecture

Each module is self-contained and documented in its header.

| Module | Files | Responsibility |
|--------|-------|----------------|
| **Screen Saver host** | `main.cpp`, `ScreenSaverHost.*` | Arg parsing, per-monitor windows, preview child window, input-to-exit, frame limiter, DPI awareness. |
| **Engine** | `Engine.*` | Per-window orchestrator; owns the clock and drives all modules each frame. |
| **Renderer** | `Renderer.*`, `shaders/*.hlsl` | D3D11 device + flip-model swap chain (tearing/VRR), HDR target, instanced draws, bloom + tonemap post chain, Direct2D interop for the HUD. |
| **Fish AI** | `Fish.*` | Autonomous boids agents: separation/cohesion/alignment, wander, edge & predator avoidance, food seeking, personality vectors, spatial-hash neighbour queries, natural enter/leave life-cycle, cyber creatures. |
| **Physics / math** | `NyxMath.h` | Vectors, steering helpers, RNG, drift noise. |
| **Particles** | `ParticleSystem.*` | Stochastic bubble vents + suspended motes, instanced point sprites. |
| **System Monitor** | `SystemMonitor.*` | Real CPU / memory / process / uptime telemetry → HUD + a synthesized threat score that nudges fish behaviour. |
| **HUD** | `Hud.*` | Direct2D/DirectWrite dashboard: brand, system health, threat monitor, clock, threat feed, fading tips. Scales with DPI. |
| **Educational tips** | `EducationalTips.*` | Rotating fact pool with fade in/out, merges fresh AI tips when available. |
| **Ollama client** | `OllamaClient.*` | Background-thread WinHTTP call to a local Ollama server; never blocks the render loop; silent offline fallback. |
| **Settings** | `Settings.*`, `SettingsDialog.*`, `resources/*` | Registry-backed config (`HKCU\Software\Nythos\CyberAquarium`) + the configuration dialog. |

### Frame pipeline
```
[HDR target]  background  ->  fish (1 instanced draw)  ->  particles (1 instanced draw)
[post]        bright-pass ->  blur H  ->  blur V
[backbuffer]  composite (ACES tonemap + bloom + water distortion + grain)
[backbuffer]  HUD (Direct2D)
Present (uncapped, tearing-allowed; main loop frame-limits to 60/144)
```

## Configuration

Settings dialog (`/c`) — persisted to the registry:

- **Ecosystem:** fish count, bubble density, threat creatures on/off
- **Presentation:** quality (Performance / Balanced / Ultra), theme
  (Reef / Abyss / Cyberpunk), lighting intensity, cyber HUD, educational tips
- **AI tips:** use local Ollama, model name (default `nythos-qwen:latest`)
- **Audio:** ambient music *(roadmap)*

The Ollama host/port default to `127.0.0.1:11434` and can be changed in the
registry. If Ollama isn't running, the built-in tip pool is used seamlessly.

## What's implemented vs. roadmap

**Implemented:** real `.scr` (all modes, multi-monitor, High-DPI), D3D11 HDR
pipeline with bloom/tonemap/water-distortion, **side-on layered reef** (depth
gradient, god rays, caustics, far hazy ridge + colourful coral mounds + swaying
kelp + sandy bed + foreground corner framing, vignette) with fish kept in the
water column above the crest, GPU-instanced boids fish with
per-pixel tail undulation & body bend and depth fog, stochastic bubbles/motes,
cyber creatures (malware fish + ransomware shark cause scatter), live system
telemetry HUD + clock, educational tips with Ollama integration, settings +
registry, frame limiter for low CPU.

**Roadmap (the spec's stretch goals):** screen-space reflections, depth of
field, motion blur, soft shadows / SSAO, audio (ocean ambience, mute-aware),
octopus/whale/dolphin creatures, distinct per-species 3D meshes, 1000+ fish via
a GPU compute-shader boids pass, sharper coral silhouettes / coral instances,
and a photoreal reef background texture option.

## Notes / gotchas

- **Don't** name a header `Math.h` with `src/` on the include path — on
  case-insensitive Windows it shadows the CRT `<math.h>` that `<cmath>` pulls in.
  (This file is `NyxMath.h` for that reason.)
- The HUD draws via `ID2D1RenderTarget` created on the swap-chain back buffer;
  it's recreated on resize alongside the D3D targets.
